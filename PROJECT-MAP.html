2. 단계별 파일 생성 맵

각 단계를 진행하면 → 어디에 어떤 파일이 생기고 무슨 내용이 담기는지.

진행 현황 (2026-06-07 세션4): M1~M2 + E4·E5a·E5b·콘솔 리디자인·고객포털·KPI 시안 완료. 이번 세션 = E5 견적 시스템 풀스택 완주(하루 4배포, superpowers TDD): ①견적 계산 엔진(#60, v0.12.3.0) — 순수함수 calculateQuote(장비/옵션 단가×수량·헤드=옵션 한 줄로 일반화·세액 round 10%), 가격표 비내장. ②채번+불변버전(#60) — JHQ-YYYYMMDD-NNN-VN 연도리셋 카운터·재발행=번호유지+version MAX+1·issued 동결(pdf_url만 예외). ③생성 결선 RPC(#61, v0.12.4.0) — create_quote/create_manual_quote(SQL DEFINER가 금액 재계산=서버권위, app source=manual+quote 원자생성)·TS==SQL 교차검증. ④견적 작성 콘솔 UI(#62, v0.12.5.0) — 작성(/admin/applications/[id]/quote/new)·수기(/admin/quotes/new)·상세+재발행(/admin/quotes/[id], ?from= 프리필→같은번호 V2), 실시간 합계. ⑤통합 PDF 워커 골격+jobs 큐(#63, v0.12.6.0) — claim_next_job SKIP LOCKED·enqueue 트리거·워커 폴링·pdf-lib placeholder PDF→quote-pdfs→pdf_url. 이어 ⑥워커 Railway 배포설정(#66) — shared가 TS소스라 node dist 불가 → tsx 런타임·railway.json. ⑦의뢰 상태 5단계(#67, v0.12.7.0) — 접수→배정→견적중(작성중)→견적발송(발행됨)→완료(종결·네이비). _quote_insert 자동 전이(draft→견적중·발행→견적발송, 앞으로만). 게이트 GREEN(web178·worker2·db-tests303·shared73·e2e44·tsc·lint·build·as any 0). ⚠️ PDF 레이아웃=placeholder(견적양식 의뢰사 대기)·Railway 워커 대시보드 실배포 미완(코드는 준비)·레이아웃 일괄정비 예정(완료 StatusControl 위치 등). 다음 = start 시: E5 코드는 거의 소진(계산·채번·생성RPC·작성콘솔·PDF골격 완료). 남은 E5 = 의뢰사 자료/인프라 대기 — ⓐ 실 견적서 레이아웃(양식 수령 시 apps/worker/src/jobs/render-quote-pdf.ts만 교체+한글폰트 임베드) ⓑ Railway 워커 배포(잡 소비 시작) ⓒ 옵션 가격표·장비 자동가격·이미지·번호형식 확정. 코드로 지금 가능: 전화 placeholder(1577-0000)·브랜드색 `--color-accent`·KPI 실집계 배선·S1 고객 엑셀 이관(컬럼 대기)·P-G 카톡알림(#25). ⚠️ db reset 후 e2e면 bash supabase/seed/seed-local.sh 시드 복구 필수. 운영이월: 홈 cold-start·고아청소 cron.
