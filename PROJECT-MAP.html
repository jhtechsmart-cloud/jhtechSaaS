2. 단계별 파일 생성 맵

각 단계를 진행하면 → 어디에 어떤 파일이 생기고 무슨 내용이 담기는지.

진행 현황 (2026-05-30): E1·E2 완료·머지·배포(이슈 #2·#3 닫힘, v0.2.0.0). 현재 E3(공개 카탈로그/상세 + 견적요청 폼) 진행 중 — 설계·플랜 done → 2개 sub-plan 분해(P1 카탈로그/상세 / P2 폼+RPC), P1 완주(subagent-driven 11태스크 TDD, 전수리뷰 READY TO MERGE, 게이트 GREEN vitest 44·tsc·lint·build, 브랜치 feat/e3-public-catalog push·미머지). 다음 = E3 P2(견적요청 폼 + submit_application RPC) → P1+P2 한 PR로 E3 머지.

아래 단계 바 = 한 phase의 생애주기. E1·E2는 ①~⑨ 완주. E3는 설계 done, ⑥구현이 P1 done·P2 진행 예정(P1+P2 한 PR 머지 전략이라 ⑦~⑨는 P2 후).

① 기획→ ② 셋업→ ③ 스펙→ ④ 검토→ ⑤ 설계/UI-SPEC→ ⑥ 구현(E3 P1 done · P2 다음)→ ⑦ 코드리뷰→ ⑧ QA→ ⑨ 배포
단계명령어생기는 파일 (위치)담기는 내용
