# 고객관리 중복 로직 + 견적요청 매칭 + 개인정보 동의 전문 — 설계

**한 문장 요약**: 고객 등록 시 "이름만 같은 회사"도 경고하고, 공개 견적요청은 관리자 화면에서 기존 고객과 자동 대조·연결·교정하게 하며, 비어 있던 개인정보 동의 전문을 법정 요건에 맞게 채운다.

**왜 필요한가**: 지금은 오타 하나로 같은 회사가 두 번 등록되고, 견적요청은 기존 고객과 아무 연결 없이 쌓이며, 개인정보 동의문이 placeholder라 법적 리스크가 있다.

날짜: 2026-07-14 · 승인: Seonje

## 배경 (현재 상태 실측)

- `check_company_duplicate` RPC(마이그 `20260713120100`)가 ①사업자번호 정확 일치 ②회사명(공백제거·소문자)+전화(숫자) 동시 일치 2단 판정. 폼 실시간(400ms 디바운스) + 서버 액션 저장 직전 재검증(fail-closed) 이중 방어. 둘 다 저장 차단.
- 회사명 **단독** 일치는 어떤 게이트에도 안 걸림 → `/admin/customers/duplicates` 수동 조회 페이지에만 노출.
- 공개 견적요청(`submit_application` RPC)은 고객 입력 원문을 `applications` 자체 컬럼에 저장. `companies` 매칭·생성 없음. `applications.company_id`는 수기견적 경로만 사용(공개 접수 건 null).
- `privacy_policies` v1.0 행의 `body` = `'[플레이스홀더 — 법무 확정 후 교체]'`. 견적·A/S·소모품 3개 공개 폼이 전부 이 행을 읽어 표시. `PRIVACY_VERSION = "v1.0"` 상수가 3개 스키마 파일에 중복.

## 결정 사항 (질의 확정)

1. 공개 견적요청의 매칭·오타 필터링 = **관리자 쪽에서 처리** (익명 조회에 의한 개인정보 노출 경로 차단).
2. 신규 고객 등록 동명(회사명만 일치) = **경고 + "동명의 다른 회사가 맞습니다" 확인 후 진행** (차단 아님).
3. 견적요청 매칭 확인 시 = **연결(company_id) + 필드별 선택 교정** (자동 덮어쓰기 없음).
4. 개인정보 보유 기간 = **목적 달성 후 3년**, 책임자 표기 = **대표 연락처만** (02-839-7723 / support@jhtech.co.kr).

## 작업 1 — 신규 고객 등록: 동명 경고

- `check_company_duplicate` RPC에 3순위 판정 추가: 회사명 정규화(공백 제거·소문자) 단독 일치 → `match_kind='name_only'` 반환. 1·2순위(저장 차단)는 불변.
- `CompanyForm`: `name_only` 매치 시 기존 고객 정보(사업자번호·담당자·주소) 경고 배너 + "동명의 다른 회사가 맞습니다" 체크박스를 켜야 저장 가능.
- 서버 액션(`createCustomer`/`updateCustomer`): 확인 플래그(`name_only_confirmed`) 없이 동명 저장 요청이 오면 거부(fail-closed). 수정 시에는 `excludeId`로 자기 자신 제외(기존 패턴 유지).
- 마이그레이션 1건(RPC CREATE OR REPLACE) + `supabase/rollback/` 롤백, db-tests + 폼/액션 단위 테스트.

## 작업 2 — 견적요청 관리 화면: 기존 고객 매칭·연결·선택 교정

- 매칭 기준: ①사업자번호 일치 → "기존 고객" 배지 ②회사명(정규화) 일치인데 사업자번호 불일치/없음 → "확인 필요" 배지.
- 목록: 배지만 표시. 목록 쿼리에서 한 번의 조인/일괄 대조로 계산(행별 추가 쿼리 금지).
- 상세: 배지 + "이 고객으로 연결" 버튼 → `applications.company_id` 세팅.
- 선택 교정: 연결 시 요청 입력값 vs 고객DB 값이 다른 필드(회사명·대표자·사업자번호·연락처·이메일·주소)를 나란히 표시, 필드별로 "요청값을 고객DB에 반영" 또는 "고객DB값으로 요청 교정" 선택. 자동 덮어쓰기 없음.
- `registerFromApplication`(견적요청→신규 고객 전환) 유지 + 전환 시 `company_id`도 연결되도록 보강(`upsert_company_from_application` 반환 id를 applications에 기록).
- 권한: 연결·요청 교정 = 로그인 관리자(해당 의뢰 접근 가능자), 고객DB 갱신 = `customers.manage`.

## 작업 3 — 개인정보 수집·이용 동의 전문

- 전문에 개인정보 보호법 제15조 필수 4요소 포함:
  - 수집 항목: 회사명, 대표자명, 사업자등록번호, 담당자 성명·연락처·이메일, 주소, 설치환경 정보(견적요청), 장비·증상 정보(A/S), 소모품 주문 정보 — 3개 폼 포괄 서술.
  - 목적: 견적·A/S·소모품 문의 상담 및 처리, 처리 이력 관리.
  - 보유 기간: 목적 달성 후 3년 보관 후 파기(거래 성사 시 관계 법령에 따라 보존).
  - 거부 권리: 동의 거부 가능, 거부 시 해당 온라인 접수 이용 제한 안내.
  - 문의처: (주)재현테크 02-839-7723 / support@jhtech.co.kr.
- 반영 방식: `privacy_policies`에 **v1.1 새 행 INSERT**(v1.0 body 불변 — 기존 동의 기록의 원문 보존) + 3개 스키마 파일 `PRIVACY_VERSION`을 `"v1.1"`로 동시 범프. 마이그레이션 + 롤백.
