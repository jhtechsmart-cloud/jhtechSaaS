# 하이웍스 메일 발송 연동 — 미팅 질문 시트

> **미팅 목적 한 줄**: 발행된 견적서 PDF를 **영업담당자 명의로 고객에게 메일 발송**하고, 그 메일이 **담당자 하이웍스 "보낸편지함"에 남게** 하는 기능을 만들려고 합니다. 이를 위한 **REST API 연동 스펙**을 확인하러 왔습니다.
>
> 미팅 날짜: 2026-06-17 · 연동 대상: 가비아 하이웍스 메일 REST API
> 현재 상태: 우리 쪽 SaaS는 견적서 PDF 자동생성까지 완료. **메일 발송만 남았고, 하이웍스 API 스펙이 없어 보류 중.**

---

## 왜 SMTP가 아니라 REST API인가 (우리가 이미 확인한 것)
- 하이웍스는 **POP만 지원, IMAP 미지원** → 외부 SMTP로 보내면 **보낸편지함에 안 남음**.
- 요구사항이 "담당자 보낸편지함에 기록"이라 → **하이웍스 자체 발송(REST API)** 외엔 방법이 없음.
- 그래서 이 미팅의 목표 = **REST API 연동에 필요한 스펙·계정·검수 절차 확보**.

## 미팅 끝나고 우리가 갈리는 길 (이해용 요약)

```
Q1 인증 모델?
 ├─ 앱 토큰 1개 + Q3 from 임의지정 OK
 │     → 단순: 토큰 1개 env에 넣고 HiworksMailSender 구현 끝
 └─ 담당자별 OAuth 필요
       → 큼: 담당자 토큰 저장 테이블 + 동의 UX + 토큰 갱신 추가
              (그래도 발송 로직 본체는 이미 다 만들어 둠)

Q2 보낸편지함 적재 안 됨?  → 요구사항 재협의 필요 (기능 전제 붕괴)
```

> 우리 쪽은 하이웍스에 의존하지 않는 부분(잡 큐·메일 본문·PDF 첨부·발송 로그·버튼)을 **가짜 발송기로 먼저 다 만들어 둘 예정**.
> 스펙 확정되면 **`HiworksMailSender` 클래스 하나만** 끼우면 됩니다.
