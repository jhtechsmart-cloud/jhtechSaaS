## 결정 사항(브레인스토밍)
1. **목표**: 견적 발행 → 견적서 PDF를 고객 이메일로 발송. **담당자 개인 명의 발송 + 담당자 하이웍스 보낸편지함에 기록** 필요(Seonje 확정).
2. **발송 경로 = 하이웍스 REST API**(SMTP 아님).
   - 이유: 하이웍스는 **POP만 지원, IMAP 미지원** → **SMTP 외부 발송은 보낸편지함에 안 남는다**. 보낸편지함 적재는 하이웍스 자체 발송(REST API)뿐.
3. **현 상태 = 스펙 미확보로 보류**. REST API는 앱 등록 + 사전 검수(영업일 ~5일)가 필요하고, 인증 모델/첨부/발송자 지정 여부가 개발자센터 콘솔(로그인)에만 있음. 추측 설계는 헛수고 위험이라 스펙 확보를 선행.

## 조사 결과(하이웍스)
- SMTP: `smtps.hiworks.com` · SSL · 465. 계정별 [메일>환경설정]에서 POP3/SMTP 사용 켜야 함.
- **IMAP 미지원(POP only)** → SMTP 발송분 보낸편지함 미적재.
- REST API: 하이웍스 "서비스 API"에 메일 발송 포함. HTTP REST. 개발자센터>API 콘솔>애플리케이션 등록>사전 검수(~5영업일).
- 문의처: 가비아 하이웍스 고객센터 1661-4370 / 개발자센터 포럼(developers.hiworks.com/support).
- 출처: customer.gabia.com/manual/hiworks/2047/2394(POP3/SMTP·IMAP미지원), developers.hiworks.com/support(앱등록·검수5일), hiworks.com/manual/hiworks/3460/3529(서비스 API).

## ⚠️ 하이웍스에 확인할 것 (스펙 확보 체크리스트)
메일 발송 REST API에 대해:
1. **인증 모델** — 앱 단위 API 키(서버 한 개 토큰)인가, **사용자 단위 OAuth**(담당자별 동의·토큰)인가? "담당자 개인 명의 발송"을 하려면 사용자별 인증이 필요한가?
2. **발송자(from) 지정** — 임의 from 지정 가능? 아니면 인증 계정으로 고정?
3. **첨부파일** — 발송 시 PDF 첨부 가능? 형식(base64/multipart)·용량 제한?
4. **보낸편지함 저장** — API로 보낸 메일이 그 사용자의 하이웍스 보낸편지함에 남는가? (핵심 요구사항)
5. **엔드포인트/요청 형식** — 발송 API URL·메서드·요청 바디 예시.
6. **승인·제한** — 앱 등록·검수 기간, rate limit, 요금.
